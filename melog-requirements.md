# MELOG 要件定義書 v1.0

> メンズエステ体験談・コラム・情報に特化した、note品質のUGCマネタイズプラットフォーム
> 作成日：2026-07-03

---

## 1. プロダクト概要

### コンセプト
メンズエステ体験談・コラム・情報に特化した、**note品質のUGCマネタイズプラットフォーム**。

ライターが記事を書いて収益を得る。読者は良質な情報にアクセスできる。プラットフォームは手数料で収益を得る。三者全員が得をする構造。

### ポジショニング
| | ワクスト | note | MELOG |
|---|---|---|---|
| コンテンツ特化 | ◯ | × | ◯ |
| UI品質 | △ | ◎ | ◎ |
| マネタイズ | ◯ | ◯ | ◯ |
| モバイル対応 | △ | ◎ | ◎ |
| エリア・店舗検索 | △ | × | ◎ |

### 収益モデル
- ライターの有料記事売上に対して **プラットフォーム手数料15%**
- ポイント購入時に手数料を内包（実質ユーザー負担）

## 2. ユーザーロール

### 2-1. ゲスト（未ログイン）
- 無料記事の全文閲覧
- 有料記事のプレビュー（冒頭500字程度）閲覧
- 検索・ブラウジング
- 会員登録への導線

### 2-2. 読者（無料会員）
- 全無料記事閲覧
- ポイント購入・有料記事購入
- ブックマーク・いいね
- ライターフォロー
- コメント投稿

### 2-3. ライター（投稿者）
- 読者の全権限 ＋
- 記事投稿・編集・削除
- 有料記事の価格設定
- 売上ダッシュボード閲覧
- 出金申請

### 2-4. 管理者
- 全コンテンツの閲覧・削除
- ユーザー管理（BAN・警告）
- 売上・手数料管理
- お知らせ投稿
- 統計ダッシュボード

## 3. 機能要件

### 3-1. 認証・会員

**登録**
- メールアドレス ＋ パスワード
- X（Twitter）OAuth ログイン
- Google OAuth ログイン
- 登録時：メール認証必須

**プロフィール**
- ユーザー名（表示名）
- アイコン画像
- 自己紹介文（500字以内）
- 活動エリア（任意）
- フォロワー数 / フォロー数
- 投稿記事一覧
- ライター申請ボタン（読者→ライター昇格）

**セキュリティ**
- パスワードリセット（メール）
- ログイン履歴確認
- アカウント削除（論理削除）

---

### 3-2. 記事エディタ

noteのエディタと同等品質を目指す。

**基本機能**
- リッチテキスト編集（Tiptap採用予定）
- 見出し H2 / H3
- 太字・斜体・下線
- リスト（箇条書き・番号）
- 水平線（区切り線）
- 引用（blockquote）
- リンク挿入
- 画像挿入（ドラッグ&ドロップ / クリップボード貼り付け）
- 文字数カウント（リアルタイム）

**有料設定**
- 全文無料 / 部分有料 / 全文有料
- 部分有料の場合：プレビュー終了位置をエディタ上でドラッグ設定
- 価格：100pt / 200pt / 300pt / 500pt / 1,000pt / カスタム

**メタ情報**
- サムネイル画像（推奨サイズ表示）
- 記事タイトル（80字以内）
- 概要文（150字以内・OGP用）
- タグ（最大5つ）
  - エリアタグ（例：新宿・渋谷・梅田）
  - サービスタグ（例：オイル・ヌキあり・四つ手）
  - 価格帯タグ（例：〜1万円・1〜2万円・2万円〜）
  - フリータグ

**下書き・予約投稿**
- 下書き保存（自動保存 30秒ごと）
- 予約投稿（日時指定）
- 公開後の編集・非公開化

---

### 3-3. 記事閲覧

**読了体験**
- 最大横幅680px（noteと同等）
- 行間・文字サイズをコンテンツ品質最優先で設定
- 見出しへのアンカーリンク
- 推定読了時間表示（例：「約5分で読めます」）
- 進捗バー（画面上部）

**有料ゾーン**
- プレビュー末尾に購入CTA
- 購入後は即全文閲覧
- 一度購入した記事は永久閲覧可能

**記事アクション**
- いいね（ハートボタン）
- ブックマーク
- X・LINEシェアボタン
- URLコピー
- 記事通報

**ライター情報**
- 記事末尾にライタープロフィールカード
- フォローボタン
- 他の記事へのリンク

### 3-4. ポイントシステム

**ポイント購入**
| パック | 価格（税込） | ポイント |
|---|---|---|
| お試し | ¥500 | 500pt |
| スタンダード | ¥1,000 | 1,050pt（5%増量） |
| まとめ買い | ¥3,000 | 3,300pt（10%増量） |
| ヘビー | ¥5,000 | 5,750pt（15%増量） |
| プレミアム | ¥10,000 | 12,000pt（20%増量） |

1pt ≒ 1円として扱う（購入時に増量分が特典になる形）

**決済手段（要検討・最重要）**
- クレジットカード（GMOペイメントゲートウェイ or Epsilon）
- コンビニ払い（Epsilon経由）
- キャリア決済 — ezweb / SoftBank / Y!mobile
- ※ Stripe / PayPal は規約上リスク高いため非採用

**ポイント有効期限**
- 最終ログインから180日

### 3-5. ライター収益

**売上の仕組み**
```
読者が 1,000pt の記事購入
→ ライターに 850pt 付与（85%）
→ プラットフォームが 150pt 取得（15%）
```

**売上ダッシュボード**
- 累計売上・月別売上グラフ
- 記事別売上ランキング
- フォロワー推移
- 記事PV数

**出金申請**
- 最低出金額：3,000pt（3,000円相当）
- 申請→審査→振込：月2回締め（15日・末日）
- 振込先：銀行口座（本人確認書類提出必須）
- 振込手数料：一律200円（ライター負担）

### 3-6. トップページ・発見機能

**トップページ構成**
```
[お知らせバー]
[ヘッダー：ロゴ / 検索 / ログイン]
[カテゴリナビ：エリア / サービス / 新着 / 人気]
──────────────────────────
[今日のピックアップ（編集部選定）]
[新着記事（グリッド）]
[今売れている記事（ランキング）]
[エリア別ブラウズ]
[人気ライター]
──────────────────────────
[フッター]
```

**エリアブラウズ**
- 都道府県 → 市区町村 の2階層
- 例：東京都 → 新宿 / 渋谷 / 池袋 / 銀座 / 錦糸町...
- 大阪・名古屋・福岡など主要都市も

**検索**
- キーワード全文検索
- エリア / サービス / 価格帯での絞り込み
- 並び替え：新着 / 人気 / 売れている順

**タグページ**
- タグ一覧ページ
- タグ別記事一覧

## 4. 非機能要件

### パフォーマンス
- トップページ LCP：2秒以内
- 記事ページ LCP：1.5秒以内
- 画像：WebP変換 ＋ 遅延読み込み
- CDN：Cloudflare

### SEO
- 記事ページはSSR（サーバーサイドレンダリング）
- OGP / Twitter Card 自動生成
- サイトマップ自動生成
- 構造化データ（Article schema）

### セキュリティ
- HTTPS必須
- CSRF対策
- SQLインジェクション対策
- 画像アップロード：ウイルススキャン ＋ ファイル形式制限
- レートリミット（ログイン試行・API）
- 18歳以上確認の入口表示（年齢確認モーダル）

### モバイル対応
- スマートフォンファースト設計
- タップターゲット最小44px
- 下部固定ナビゲーション（スマホ）

### 可用性
- 稼働率目標：99.5%以上
- バックアップ：DB日次スナップショット

## 6. データモデル（主要テーブル）

```sql
users
  id, email, password_hash, display_name, avatar_url,
  bio, area, role(reader/writer/admin),
  created_at, updated_at

articles
  id, user_id, title, body(HTML), summary,
  thumbnail_url, status(draft/published/private),
  price_pt(0=無料), preview_position,
  published_at, view_count, like_count,
  created_at, updated_at

tags
  id, name, slug, category(area/service/price/free)

article_tags
  article_id, tag_id

purchases
  id, user_id, article_id, pt_spent, created_at

points_transactions
  id, user_id, type(purchase/earn/spend/expire),
  amount, balance_after, ref_id, created_at

withdrawal_requests
  id, user_id, amount_pt, status(pending/approved/rejected/paid),
  bank_info(暗号化), requested_at, processed_at

follows
  follower_id, followee_id, created_at

likes
  user_id, article_id, created_at

bookmarks
  user_id, article_id, created_at

comments
  id, user_id, article_id, parent_id, body,
  status(visible/hidden), created_at

notifications
  id, user_id, type, ref_id, read_at, created_at
```

## 7. 開発フェーズ

### Phase 1 — MVP（約2〜3ヶ月）
- 会員登録・ログイン（メール＋OAuth）
- 記事投稿・閲覧（無料のみ）
- タグ・エリア検索
- プロフィールページ
- フォロー・いいね・ブックマーク
- トップページ（新着・人気）

→ **まずコンテンツと読者を集める**

### Phase 2 — マネタイズ（＋1〜2ヶ月）
- ポイント購入（決済連携）
- 有料記事設定
- ライター売上ダッシュボード
- 出金申請フロー

→ **収益化開始**

### Phase 3 — 成長機能（＋1〜2ヶ月）
- コメント機能
- 通知システム
- メールマガジン
- 編集部ピックアップ
- 管理画面の充実
